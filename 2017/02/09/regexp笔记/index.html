正则表达式

创建

1. 使用构造函数创建

    1) 括号内传入字符串，字符串需要先被解析，再处理成正则表达式
    2) 'ab\nc' => ab (换行) c 
    3) 字符串中的反斜杠表示转义

2. 使用直接量创建

1) 斜杠里传入正则表达式
2) /ab\nc/ => /ab\nc/ 
3) 字符串中的反斜杠不需要处理

[]

1. 表示匹配一个字符
2. 一些特殊字符放到[]里，在某些情况下需要转义，
    比如 - ：
    可以表示连字符(1999-2-3) 需要转义
    也可以表示范围(0-9) 不需要转义
3. [0-9]  => 0到9中任意一个字符

{}

1. 用于表示具体的重复次数
2.  {2} 重复两次
    {2,5} 重复2到5次
    {2,} 重复2次及以上
3. + * ? 也可以用来表示重复
4.  + : 1次及以上 [1,∞]
    * : 0次及以上 [0,∞]
    ? : 1次及以下 [0,1]

()

1. 用于包裹必须同时满足匹配的多个字符，即一组字符
2. 这组字符被视为一个整体

常用字符

1. \d 数字字符 
2. \s 空白字符(space tab newline 等类似符号)
3. \w 数字、字母、下划线
4. \D 非数字
5. \S 非空白
6. \W 非数字、字母、下划线 
7. .  非回车
8. i  大小写不敏感，放在正则表达式结尾/后面 如 /w/i
9. g  全局内匹配
10. ^  取反，不能包含^后面的字符 如 [^123] 表示除了 1 2 3 以外的其他字符

test函数 是 正则表达式的函数

1. objReg.test(objStr)
    objReg 必选项 RegExp对象名称 
    objStr 要进行匹配检测的字符串 
    返回 true 或 false

match函数、exec函数

exex函数 是 正则表达式的函数

match函数 是 字符串对象的函数

    1. stringObj.match(rgExp)

    • stringObj为字符串

    • rgExp为正则表达式必选项

    • 返回 结果数组 或者 null

      • 结果数组中添加了普通数组没有的index 和 input 属性

    • 如果正则表达式以g结尾，则会忽略捕获分组

    • 默认情况下，没有g结尾，则只会匹配一次，在有分组的情况下输出分组结果

    1. objReg.exec(string)

    • objReg，RegExp对象的名称

    • string，要进行匹配的字符串

    • 返回 结果数组 或者 null

    1. 两个函数的调用对象和参数互反
    1. 在正则中出现()包裹的子字符组，则先返回全部匹配结果，再返回子组匹配结果
    1. 若整个串没有匹配，返回 null，若子组没有匹配成功，返回 undefined
    1. match函数正则表达式若包含全局g，则不再匹配子串
    1. exec函数匹配第一个成功后，立即返回

选择

1. 使用 | ( 或运算符 垂直线) 表示选择关系
2. 根据需要选择是否将两个分支用()包起来
3. 选择匹配在某些情况下会引起回溯 ( *或+等重复操作符经常发生回溯)
4. 选择匹配的分支之间应该是有顺序的，当某一个分支可以匹配结束时，整个匹配结束，即只返回第一个匹配结果

二进制分支: [01]+b
十进制分支: \d+
十六进制分支: [\da-f]+h
特殊情况：100010101101010 在二进制判断到尾部才能发现这个数并不满足二进制
          212334445566665h 同样到最后才发现并不是十进制

replace函数

replace函数 是 字符串对象的函数

  1. stringObj.replace(rgExp, replaceText)

    • stringObj是字符串(string)
    • reExp可以是正则表达式对象(RegExp),也可以是字符串(string)
      replaceText是替代查找到的字符串，也可以是一个自定义函数,函数的返回值- 要插入的新字符串
    • 返回新的字符串
    • 默认只进行一次匹配，遇到第一个位置

  2. 如果第一个参数正则表达式中含有()分组，在第二个参数中，可以用$&来引用整个串，用$1, $2 … $9来引用各个子组返回的结果(匹配元组)

  3. 第二个参数也可以为一个函数,每次匹配都调用函数并以匹配元组(或匹配整体)作为参数，返回需要插入的新字符串。

贪婪匹配

1. 贪婪: 重复运算符(+、*、?、{})会尽可能多的匹配它们可以匹配的字符，然后回溯
2. 懒惰: 在上述字符后面添加一个 ? 可以使其尽可能少的匹配字符

边界

1. 表示字符( \w 可以匹配的字符--包括数字、字母、下划线_ ) 与非字符相接的位置，并不是一个实际字符
  1. ^、$ 分别用来表示匹配串的首尾边界
  2. \b 用来匹配一个单词的边界
  3. /^/ ^ 仅表示开头位置，可以匹配任何，没什么意义，$ 也是一样的情况
    /[^]/ ^ 表示取反， 可以匹配任何非空字符
    /[$]/ $ 表示$符号 /[$]/ === /\$/

    零宽断言

  4. 断言: 满足一定条件即断言，用来声明一个应该为真的事实
  5. 用于查找在某些内容(但不包括)之前的东西，即指定某个位置(仅仅是位置，没有实体字符)
  6. js仅支持先行断言 (?=exp),即选取满足exp条件之前的字符
