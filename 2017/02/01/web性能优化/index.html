web性能优化

发表于 2017-02-01 | 分类于 知识梳理 |

网络层

  • 减少http请求
    • 合并请求
    • CSS sprite
  • 使用CDN(Content Delivery Network)
    • 6-8，一般情况下同一个域名下同时只能建立6-8个连接
    • 解决办法：增加浏览器同时请求资源的数量
      • 可以把资源放在不同的服务器上(域名)
      • 针对不同地区的人使用不同的服务器(DNS服务器根据USER自身的IP地址分配一个离他最近的服务器IP地址)
  • 使用缓存
    • 各种不同的缓存类型
      • IF -Modified-Since
      • Last-modified
      • Expiress
      • Cache-Control
  • 压缩
    • 在HTTP连接上使用gzip压缩

页面层

  • 将CSS放在页面的顶部
    • 把CSS直接内联到页面里
    • 只渲染首屏CSS
    • 多段式Link标签
  • 将JS放在底部
    • js的下载和运行都会阻塞、暂停页面的解析和渲染
    • 解析：解析HTML并构建DOM树
    • 渲染：把页面画在页面上
    • 阻塞的原因
      • 单线程
      • 运行js和渲染页面是同一个线程
      • js的代码有可能是操作DOM的，所以必须等DOM渲染出来
  • 避免CSS表达式
  • 使用外部JS和CSS文件
    • 因为多个页面使用同一个文件时，缓存会起作用
  • 分散域名
  • 提前解析DNS ，dns-prefetch
  • 精简js
    • 混淆，minifiy
      • 去掉多余的空格，注释，换行，混淆代码
      • Uglifyjs
      • 减少体积，保护知识产权
  • 避免重定向
    • 响应头：
    • 避免30X请求，除缓存
  • 移除重复脚本
  • 配置Etag
    • Etag是一个http头，类似于文件hash
    • 一般来说如果服务器检测到相同，就可以回复一个304请求
    • 不同的机器，HTTP服务器(apache)的默认配置
      • 会导致相同的文件在 不同的机器上返回不同的Etag头
      • 是ajax可缓存

  • 把资源放在localStorage里

    • imququ.com

  • HTTP2

    • iHover -> Youtube -> HTTP2视频
    • 强制要求走HTTPS
      • HTTPS需要证书
      • 证书需要找相应的CA机构去买
